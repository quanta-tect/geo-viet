Analytics

Cách cài GA4 cho site GEO

Nên thu thập analytics để biết traffic đến từ đâu, bài nào được người dùng quan tâm và kênh nào mang lại cơ hội cite tốt nhất.

Vì sao site GEO vẫn nên có analytics?

ChatGPT Search, AI visibility và GEO cần dữ liệu thực tế để tối ưu. Bạn không chỉ cần biết bài được cite, mà còn cần biết:

  • nguồn referral: chatgpt.com, perplexity.ai, bing.com, google.com
  • bài nào giữ người đọc lâu nhất
  • kênh nào chuyển đổi tốt

GA4 dùng để đo gì?

  • Pageview: bài nào được xem nhiều.
  • Source / Medium: traffic đến từ đâu.
  • Referral path: trang nào đang dẫn traffic về.
  • Event: click CTA, mở tool, scroll sâu.

Cách tạo property GA4

  1. Vào GA4 và đăng nhập Google.
  2. Chọn Admin → Create property.
  3. Đặt tên property theo tên site, ví dụ: GEO Việt Nam.
  4. Chọn Web làm platform.
  5. Điền URL site, ví dụ: https://quanta-tect.github.io/geo-viet/.
  6. Chọn Enhanced Measurement bật để tự đo pageview.
  7. Lưu, GA4 sẽ sinh Measurement ID dạng G-XXXXXXXXXX.

Cách lấy Measurement ID

Sau khi tạo property, vào Admin → Data Streams → chọn stream vừa tạo. Bạn sẽ thấy Measurement ID ở đầu trang.

Cách gắn script vào site tĩnh

Vì site này là GitHub Pages tĩnh, bạn gắn GA4 trực tiếp trong HTML:

<!-- Google tag (gtag.js) -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXXXXX');
</script>

Thay G-XXXXXXXXXX bằng Measurement ID thực tế.

Lưu ý: hiện site đang dùng Plausible placeholder. Bạn có thể chuyển sang GA4 hoặc giữ Plausible tùy chiến lược. Chỉ dùng một tool analytics chính để tránh trùng lặp dữ liệu.

Nên theo dõi event/chỉ số nào?

  • Pageview: tự động theo dõi.
  • Scroll depth: thấy người đọc đến đâu.
  • File download: nếu bạn thêm PDF checklist.
  • Outbound link: click sang OpenAI docs, Bing...
  • CTA click: nút mở tool / checklist.

Cách nhận biết traffic từ ChatGPT / Perplexity / Bing / Google

Trong GA4, vào Reports → Acquisition → Traffic acquisition. Tìm cột Session source / medium:

  • ChatGPT Search: thường hiển thị referral từ chatgpt.com.
  • Perplexity: referral từ perplexity.ai.
  • Bing: source bing hoặc microsoft.com.
  • Google: source google.

Nếu bạn thấy referral từ domain AI search, đó là tín hiệu site đang được cite. Khi đó bạn có thể lặp lại format bài đang hoạt động.

FAQ

Site GEO có cần GA4 không?

Nên có. Analytics giúp đo traffic nguồn, biết bài nào hiệu quả và tối ưu dựa trên dữ liệu thực tế.

Ga4 có đo traffic từ ChatGPT Search không?

Có, nếu ChatGPT Search trỏ người dùng về site, bạn sẽ thấy referral session tương ứng.

Nên dùng Plausible hay GA4?

Nếu muốn nhẹ, không cookie banner phức tạp, hãy dùng Plausible. Nếu cần segmentation sâu, hãy dùng GA4.

Đặt analytics ở đâu trong site tĩnh?

Chèn script vào mỗi file HTML trong <head>, ngay sau stylesheet. Đã có placeholder rõ trong các file hiện tại.
